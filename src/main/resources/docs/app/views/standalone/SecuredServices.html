Create GServ Config

A GServ Config file is a regular JSON formatted file. Below is an example of a minimal config file: 

{
    "apps" : [{
            "https": false,
            "name" : "Secured App Example",
            "resourceScripts" : ["resources/Resource1.groovy","resources/Resource2.groovy"],
            "port" : 11111,
            "static_roots" : [ "/Users/lcollins/src/main/resources/docs" ]
        }]
}
The GServ config is required to have at least one application defined. The 'apps' element is a List of Application Definitions. Each entry in this list is separate App instance that will run on its own port. Port must be unique across app instances. Application Definitions can contain the following:
  • https: boolean
  • name: String
  • resourceScripts(optional) : List<String> containing REST resource script file names
  • port: Integer port on which to listen for requests.

    • Creating REST Resources

    REST Resources Scripts define one or more Resources: 

    
import io.github.javaconductor.gserv.*

[
        GServ.Resource("/hello") {
            get("/") {
                write("Hello World".bytes)
            }

            get("/:name") { name ->
                //println l
                write("Hello $name".toString().bytes)
            }

        }
]
